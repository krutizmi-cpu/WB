# WB Unit Economics v2

Проект для расчёта юнит-экономики Wildberries с нуля:
- массовая загрузка товаров через Excel;
- массовая выгрузка результата в Excel;
- автоматическое определение категории WB по названию;
- отдельные параметры по системе налогообложения;
- отдельные параметры по СПП, рекламе, выкупаемости, браку/потерям;
- обратная логистика, зависящая от выкупаемости;
- поддержка FBS и FBW/FBO;
- мгновенный пересчёт при изменении любого параметра на экране.

## Что поменяно относительно первой версии

1. Налоги вынесены отдельно в интерфейс.
2. СПП вынесено отдельно.
3. Выкупаемость вынесена отдельно.
4. Брак вынесен отдельно.
5. Обратная логистика считается отдельно и зависит от невыкупа.
6. Для FBW/FBO добавлено плановое хранение на складе WB.
7. Пересчёт выполняется немедленно при изменении параметров.
8. Входной файл остаётся простым: артикул, наименование, габариты. Дополнительные поля можно не заполнять.

## Структура

- `app.py` — Streamlit-приложение
- `config.json` — дефолтные параметры и налоговые режимы
- `data/category_patterns.json` — словарь автоопределения категорий
- `data/wb_commissions.xlsx` — локальный справочник комиссий по категориям
- `templates/wb_template.xlsx` — шаблон загрузки
- `results/` — папка для результатов, если понадобится локально сохранять файлы

## Как запустить

```bash
pip install -r requirements.txt
streamlit run app.py
```

## Как считать

### FBS
- хранение = 0;
- логистика WB берётся как базовый тариф × коэффициент FBS;
- обратная логистика зависит от выкупаемости.

### FBW/FBO
- логистика считается по базовому тарифу WB;
- хранение = объём в литрах × ставка хранения × план хранения в днях;
- если в Excel заполнена колонка `Дней хранения FBW/FBO (опц.)`, она имеет приоритет над глобальным параметром.

## Допущения модели

### 1) Цена
Если в Excel нет колонки `Цена продавца до СПП, ₽ (опц.)`, берётся дефолт из `config.json` / интерфейса.

### 2) Себестоимость
Если в Excel нет колонки `Себестоимость, ₽ (опц.)`, берётся дефолт из `config.json` / интерфейса.

### 3) СПП
В модели рассчитывается:
- цена продавца после скидки продавца;
- цена покупателя после СПП.

### 4) Обратная логистика
Нормализована на 1 успешную продажу:
- чем ниже выкупаемость, тем выше нагрузка обратной логистики на успешную продажу.

### 5) Брак / потери
Нормализованы на 1 успешную продажу и считаются как доля от себестоимости.

## Почему справочник комиссий вынесен в Excel

Комиссии и условия WB меняются. Поэтому я не зашивал их жёстко в код:
- категории определяются автоматически;
- ставка комиссии подтягивается из `data/wb_commissions.xlsx`;
- файл можно обновлять без переписывания приложения.

## Важное замечание

С высокой точностью прибыльность без цены и себестоимости посчитать нельзя.
Поэтому шаблон сохранён простым, но:
- минимально обязательно: артикул, наименование, длина, ширина, высота;
- для точного P&L лучше заполнять ещё себестоимость и цену.
